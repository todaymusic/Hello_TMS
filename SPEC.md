## 1. 전체 구조

슬랙과 유사한 사이드바 기반 네비게이션 + 메인 콘텐츠 영역 레이아웃을 사용한다.

```
┌─────────────────────────────────────────────────────────┐
│  사이드바                        메인 영역              │
│  ┌──────────┐  ┌──────────────────────────────────────┐ │
│  │ 🏠 대시보드 │  │                                      │ │
│  │ 📅 캘린더  │  │     선택된 탭의 콘텐츠 영역           │ │
│  │ 📁 프로젝트 │  │                                      │ │
│  │ 👤 내활동  │  │                                      │ │
│  │ ─────────│  │                                      │ │
│  │ 💬 DM/채팅 │  │                                      │ │
│  │ ⚙️ 설정   │  │                                      │ │
│  └──────────┘  └──────────────────────────────────────┘ │
│  [내 상태: 🟢 업무중 ▾]                                  │
└─────────────────────────────────────────────────────────┘
```

### 핵심 탭 구성

| 탭 | 역할 |
|---|---|
| **대시보드** | 실시간 업무현황, 업무 부여, 내 상태 변경 |
| **캘린더** | 월별 업무 진행 현황, 프로젝트 타임라인, 이전 태스크 검색 |
| **프로젝트** | **프로젝트 생성·설정(담당자/참여자 추가)**, 프로젝트별 페이지, 소통 스레드, AI 요약, 아카이브 |
| **내 활동** | 오늘 할일 체크리스트, 멘션/알림 피드, 내 스케줄 타임라인, 근무 현황 |

## 2. 탭별 상세 설계

### 🆕 업무 대분류 — 롱 / 쇼츠 / 프로젝트

모든 업무는 가장 먼저 **3가지 대분류** 중 하나로 구분된다. 이 대분류는 업무의 생성 경로와 협업 방식을 결정한다.

| 대분류 | 아이콘 | 성격 | 생성 경로 | 담당 구조 |
|---|---|---|---|---|
| **롱** | 🎬 | 장편 콘텐츠 업무 | 대시보드 [업무 부여]에서 직접 부여 | 단일 담당자 |
| **쇼츠** | ⚡ | 숏폼 콘텐츠 업무 | 대시보드 [업무 부여]에서 직접 부여 | 단일 담당자 |
| **프로젝트** | 📁 | 협업형 프로젝트 | **프로젝트 탭에서 먼저 생성·설정** | 담당자 + 참여자 (다인) |

```
                  ┌──────────── 업무 대분류 ────────────┐
                  │                                      │
       ┌──────────┴──────────┐              ┌────────────┴────────────┐
       │   롱 🎬 / 쇼츠 ⚡    │              │       프로젝트 📁        │
       │  (개별 콘텐츠 업무)  │              │     (협업형 프로젝트)    │
       └──────────┬──────────┘              └────────────┬────────────┘
                  ▼                                      ▼
        대시보드 [업무 부여]에서                프로젝트 탭에서 먼저 생성
        담당자 지정해 바로 부여                  → 담당자/참여자 추가
                                                → 내부 태스크로 분배
```

**핵심 원칙**

- **롱·쇼츠**: 대시보드 [업무 부여] 패널에서 담당자를 지정해 즉시 부여하는 개별 업무.
- **프로젝트**: 대시보드에서 즉석 부여할 수 없다. 반드시 **프로젝트 탭에서 프로젝트를 먼저 생성·설정**하고, 그 안에서 담당자와 참여자를 추가한 뒤 태스크를 분배한다.
- 대분류는 캘린더 셀, 체크리스트, 카드 등에서 아이콘으로 함께 표기되어 한눈에 구분된다.

> 대분류(롱/쇼츠/프로젝트)와 산출물 요구(보고링크/설명영상)는 별개 축이다. 예: "쇼츠 + 설명영상 필요" 처럼 조합된다.

### 2-1. 대시보드 (Dashboard)

대시보드는 3개 섹션으로 구성된다.

```
┌───────────────────────┬──────────────────┬──────────────────┐
│  A. 실시간 업무현황    │  B. 업무 부여     │  C. 내 상태 변경  │
└───────────────────────┴──────────────────┴──────────────────┘
```

#### A. 실시간 업무현황 패널

팀원 카드가 리스트 또는 그리드로 배치되며, 각 카드에는 다음 정보가 표시된다.

| 항목 | 설명 |
|---|---|
| 상태 아이콘 | 🟢 온라인 / 🟡 자리비움 / 🔴 방해금지 / ⚪ 오프라인 |
| 현재 태스크 | 현재 진행 중인 태스크명 |
| 진행률 | 프로그레스 바 (0~100%) — **경과별 색상 코딩 적용** (아래 참조) |
| 프로젝트 태그 | 소속 프로젝트 라벨 |

- WebSocket을 통해 **실시간 갱신**된다.
- 🆕 **팀원 카드 클릭 시 해당 직원의 [업무 상세 화면]으로 진입**한다. 단순 태스크 목록 펼침이 아니라, 직원 단위의 상세 페이지로 전환된다.

##### 🆕 진행률 색상 코딩 (경과별)

진행률 프로그레스 바와 % 라벨은 진행 경과에 따라 색상이 단계별로 달라진다.

| 진행률 | 색상 | 의미 |
|---|---|---|
| 0% | ⚪ 회색 (`#9ca3af`) | 시작 전 / 대기 |
| 1 ~ 25% | 🔴 빨강 (`#dc2626`) | 초기 단계 |
| 26 ~ 50% | 🟠 주황 (`#ea580c`) | 진행 중 (초중반) |
| 51 ~ 75% | 🟡 노랑 (`#eab308`) | 진행 중 (중후반) |
| 76 ~ 99% | 🔵 파랑 (`#2563eb`) | 마무리 단계 |
| 100% | 🟢 초록 (`#16a34a`) | 완료 |

> 마감일이 지났는데 100% 미만인 경우 `지연(⚠️)` 상태로 표기되며, 막대 테두리가 빨강으로 강조된다.

##### 🆕 업무 상세 화면 (직원 클릭 시 진입)

| 영역 | 내용 |
|---|---|
| 직원 헤더 | 이름, 부서, 현재 상태, 오늘 누적 업무시간 |
| 진행 중 태스크 | 현재 태스크, 진행률(색상 코딩), 시작 시각 |
| 태스크 목록 | 담당 태스크 전체 (상태/우선순위/마감일별 정렬) |
| 업무 산출물 | 태스크별 보고링크 · 설명영상 · AI 업무설명 doc 모아보기 |
| 오늘 타임라인 | 해당 직원의 시작/종료 로그 기반 타임라인 |

#### B. 업무 부여 패널

| 필드 | 타입 | 설명 |
|---|---|---|
| 🆕 업무 대분류 | 선택 (롱 / 쇼츠 / 프로젝트) | 최상위 분류. **프로젝트 선택 시 직접 부여 불가 → 프로젝트 탭으로 안내** |
| 담당자 | 드롭다운 (멀티 선택) | 태스크를 부여할 직원 선택 |
| 태스크 제목 | 텍스트 | 업무 제목 입력 |
| 🆕 업무 종류 | 선택 (4종) | 산출물 요구사항 분류 (아래 참조) |
| 마감일 | DatePicker | 마감 기한 설정 |
| 우선순위 | 선택 | 긴급 / 높음 / 보통 / 낮음 |
| 프로젝트 연결 | 드롭다운 | 연결할 프로젝트 선택 (선택사항) |
| 🆕 상세 설명 | 리치 텍스트 에디터 (종류별 스위칭) | 업무 종류에 따라 설명 템플릿/필드가 전환됨 |

- 부여 즉시 담당자에게 **실시간 알림**이 전송된다.

##### 🆕 0) 업무 대분류 선택 — 롱 / 쇼츠 / 프로젝트

업무 부여 시 가장 먼저 대분류를 고른다.

```
업무 대분류:  [ 🎬 롱 ]  [ ⚡ 쇼츠 ]  [ 📁 프로젝트 ]
```

| 선택 | 동작 |
|---|---|
| 🎬 롱 / ⚡ 쇼츠 | 담당자 지정 후 이 패널에서 **바로 부여** |
| 📁 프로젝트 | 이 패널에서 부여하지 않음 → **"프로젝트 탭에서 생성하기" 버튼**으로 이동. 프로젝트 탭에서 생성·설정 후 담당자/참여자를 추가한다 (2-3 참조) |

> 프로젝트는 협업·다인 구조이므로 단발성 부여가 아니라 프로젝트 단위 설정을 거친다.

##### 🆕 1) 업무 종류 (산출물 요구사항 4분류)

업무를 부여할 때 담당자가 무엇을 제출해야 하는지를 4가지 중에서 선택한다.

| 종류 | 아이콘 | 요구 산출물 | 비고 |
|---|---|---|---|
| 보고링크 필요 | 📊 | 업무 진행 보고 링크 | 노션/문서/대시보드 등 링크 |
| 설명영상 필요 | 🎥 | 업무 설명 영상 | 화면녹화·데모 영상 링크/파일 |
| 둘 다 필요 | 📊🎥 | 보고링크 + 설명영상 | 둘 다 제출해야 완료 처리 |
| 둘 다 불필요 | — | 없음 | 산출물 없이 상태 변경만으로 완료 |

- 선택된 종류에 따라 **태스크 완료 조건**이 달라진다. (예: "둘 다 필요" 선택 시 두 산출물이 모두 등록되어야 완료 처리 가능)
- 담당자의 [오늘 할일 체크리스트]에서 종료 시, 요구 산출물 입력 폼이 종류에 맞춰 자동 노출된다. (2-4 참조)

##### 🆕 2) 업무 설명 — 종류별 스위칭

상세 설명 입력 영역은 위에서 선택한 업무 종류에 따라 **필드와 가이드가 전환**된다.

```
[업무 종류: 📊 보고링크 필요]
 ┌──────────────────────────────────────┐
 │ 업무 설명 (리치 텍스트)               │
 │ + 기대 보고 형식 안내 필드            │
 │   (예: "주차별 진행률 포함")          │
 └──────────────────────────────────────┘

[업무 종류: 🎥 설명영상 필요]
 ┌──────────────────────────────────────┐
 │ 업무 설명 (리치 텍스트)               │
 │ + 영상에 담아야 할 항목 체크 가이드   │
 │   (예: "결과 시연 / 코드 설명")       │
 └──────────────────────────────────────┘

[업무 종류: — 둘 다 불필요]
 ┌──────────────────────────────────────┐
 │ 업무 설명 (리치 텍스트)               │
 │ (산출물 안내 필드 숨김)               │
 └──────────────────────────────────────┘
```

##### 🆕 3) AI 업무설명 doc 자동 생성

부여자가 업무를 **간략히** 적으면, 프롬프트를 거쳐 AI가 정돈된 업무설명 doc을 생성한다.

```
부여자: 업무 간략 입력
   예) "랜딩 페이지 반응형 작업, 모바일 우선, 7/3까지"
              │
              ▼
   ┌─ AI 정리 프롬프트 ────────────────┐
   │ (기본 프롬프트 + 부여자 수정 가능) │
   └────────────────┬───────────────────┘
                    ▼
   AI가 업무설명 doc 생성
   (배경 · 목표 · 작업 범위 · 산출물 · 체크포인트)
              │
              ▼
   ┌─ 부여자에게 전달 (검토 단계) ──────┐
   │ • 생성된 doc 미리보기                │
   │ • 업무 설명 직접 수정 가능           │
   │ • 정리 프롬프트도 수정 후 재생성 가능 │
   └────────────────┬───────────────────┘
                    ▼
   확정 → 태스크에 doc 첨부하여 담당자에게 부여
```

| 항목 | 설명 |
|---|---|
| 입력 | 부여자가 적은 간략한 업무 메모 |
| 프롬프트 | 기본 정리 프롬프트 제공, **부여자가 직접 수정 가능** |
| 출력 | 정돈된 업무설명 doc (배경/목표/범위/산출물/체크포인트 구조) |
| 전달 | 생성된 doc은 먼저 **부여자에게 전달되어 검토**, 수정 후 확정 |
| 수정 | 업무 설명 본문 + 정리 프롬프트 **둘 다 수정 가능**, 재생성 지원 |
| 결과물 | 확정된 doc은 태스크에 첨부되어 담당자에게 전달됨 |

> AI 생성 doc은 `Task.aiDescriptionDoc`, 사용된 프롬프트는 `Task.descriptionPrompt`에 저장된다. (4장 데이터 모델 참조)

- 부여 즉시 담당자에게 **실시간 알림**이 전송되며, 첨부된 업무설명 doc도 함께 전달된다.

### 2-2. 캘린더 (Calendar)

#### 월별 뷰

```
┌────────────────────────────────────────────────┐
│            ◀  2026년 6월  ▶       [검색 🔍]     │
├──────┬──────┬──────┬──────┬──────┬──────┬──────┤
│  월  │  화  │  수  │  목  │  금  │  토  │  일  │
├──────┼──────┼──────┼──────┼──────┼──────┼──────┤
│  1   │  2   │  3   │  4   │  5   │  6   │  7   │
│김:✅ │      │이:📝 │      │      │      │      │
│박:🔄 │      │      │      │      │      │      │
```

### 2-3. 프로젝트 (Projects)

#### 프로젝트 목록 화면

```
┌──────────────────┬──────────────────┬───────────────────┐
│ 🟢 웹 리뉴얼      │ 🟢 앱 v2.0       │ 🔵 + 새 프로젝트  │
│ 6/1~6/20         │ 6/5~6/30         │                   │
│ 담당: 김,이       │ 담당: 박,최       │                   │
└──────────────────┴──────────────────┴───────────────────┘
┌─ 아카이브 ─────────────────────────────────────────────┐
│ 📦 레거시 마이그레이션 (5/1~5/25) ✅                     │
└────────────────────────────────────────────────────────┘
```

- **활성 프로젝트**: 카드 형태로 노출 (프로젝트명, 기간, 담당자)
- **아카이브**: 종료된 프로젝트는 아카이브 섹션으로 이동. 목록에서는 숨겨지지만 **검색으로 접근 가능**

#### 🆕 새 프로젝트 생성 · 설정 (담당자 / 참여자 추가)

프로젝트 대분류 업무는 **반드시 이 탭에서 먼저 생성·설정**한다. `+ 새 프로젝트` 클릭 시 설정 마법사가 열린다.

```
[+ 새 프로젝트]
      │
      ▼
┌─ STEP 1. 기본 설정 ──────────────────┐
│ • 프로젝트명 / 개요(한 줄)            │
│ • 기간 (시작일 ~ 종료일)             │
│ • 업무 설명                          │
└──────────────┬───────────────────────┘
               ▼
┌─ STEP 2. 인원 구성 ──────────────────┐
│ • 담당자 추가 (역할별: 리드/디자인/…) │
│ • 참여자 추가 (협업 멤버)            │
└──────────────┬───────────────────────┘
               ▼
┌─ STEP 3. 관련 데이터 ────────────────┐
│ • 피그마·기획서·API 명세 링크/파일    │
└──────────────┬───────────────────────┘
               ▼
   프로젝트 생성 → 상세 페이지로 진입
   → 내부 태스크 분배(칸반) 시작
```

**🆕 담당자 vs 참여자**

| 구분 | 역할 | 권한 |
|---|---|---|
| **담당자 (Owner)** | 역할별 책임자 (리드 / 디자인 / 개발 등) | 태스크 분배, 프로젝트 설정 변경, 아카이브 |
| **참여자 (Participant)** | 협업 멤버 | 스레드 소통, 태스크 수행, 산출물 등록 |

- 담당자·참여자는 프로젝트 생성 후에도 상세 페이지에서 언제든 **추가/제거**할 수 있다.
- 추가된 인원에게는 초대 알림이 전송된다.

#### 프로젝트 상세 페이지

프로젝트 클릭 시 진입하는 상세 페이지는 3개 영역으로 구성된다.

##### 📌 고정 상단 영역 (항상 노출)

프로젝트 생성 시 **필수 입력** 항목이며 스크롤해도 항상 상단에 고정된다.

| 항목 | 설명 |
|---|---|
| 프로젝트 개요 | 프로젝트에 대한 한 줄 요약 |
| 기간 | 시작일 ~ 종료일 |
| 업무 설명 | 프로젝트 상세 업무 내용 |
| 담당자 맵핑 | 역할별 담당자 (리드, 디자인, 개발 등) |
| 🆕 참여자 | 협업 멤버 목록 (역할 없이 참여) |
| 관련 데이터 | 피그마, 기획서, API 명세서 등 외부 링크/파일 첨부 |
| 진행률 | 전체 태스크 기반 자동 계산 프로그레스 바 |

##### 💬 프로젝트 커뮤니케이션 스레드

해당 프로젝트 관련 **모든 소통**이 이루어지는 공간이다.

**지원 기능:**

- 멘션 (`@이름`)
- 파일 첨부
- 코드 블록
- 리액션 (이모지)
- 답글 스레드

##### 🤖 AI 소통 요약

프로젝트 대화 내용을 AI가 자동으로 요약하여 상단에 표시한다.

**트리거 조건:**

| 조건 | 설명 |
|---|---|
| 메시지 누적 | 새 메시지 20건 이상 쌓일 때 |
| 정기 갱신 | 매일 오전 9시 자동 실행 |
| 수동 실행 | "지금 요약" 버튼 클릭 |

**요약 포맷:**

```
1) 핵심 결정사항
2) 진행 상황
3) 미결 이슈 / 액션아이템
```

##### 📋 태스크 보드

칸반 보드 형태로 프로젝트 내 태스크를 관리한다.

- 드래그 앤 드롭으로 상태 변경
- 각 태스크 카드: 담당자, 우선순위, 마감일 표시

### 2-4. 내 활동 (My Activity)

🆕 내 활동 탭은 **오늘 할일 체크리스트를 최상단**에 두고, 그 아래에 좌우 패널(멘션 피드 / 스케줄 타임라인), 하단에 근무 현황을 배치한다.

```
┌──────────────────────────────────────────────────┐
│ ✅ 오늘 할일 체크리스트  (최상단)                  │
├───────────────────────────┬──────────────────────┤
│ 📨 멘션 & 소통 피드        │ 📅 내 스케줄 타임라인 │
├───────────────────────────┴──────────────────────┤
│ 🕘 근무 현황 (업무시간 · 출퇴근 · 휴일 · 연차)     │
└──────────────────────────────────────────────────┘
```

#### 📌 개인 메모 (자동 저장 · 서버 보관)

내 활동 우측 상단에 개인용 자유 메모(포스트잇) 칸을 둔다. **본인만 보이며**, 내용은 **서버(DB)에 저장**되어 브라우저 캐시 삭제·재설치·다른 기기에서도 유지된다(기기 간 동기화).

- 저장 위치: `User.scratchMemo`(텍스트, nullable) — 사용자당 1개.
- 저장 방식: **자동 저장**. 타이핑이 멈추면 **약 0.8초 디바운스** 후 서버에 저장(`PATCH /auth/memo`), 추가로 **입력칸 이탈(blur)·탭 숨김/이탈 시**에도 저장하여 유실을 막는다. 저장 상태를 "저장됨 ✓"으로 표시.
- 불러오기: 진입 시 `GET /auth/memo`로 서버 값 로드. 서버 지연·오프라인 대비 **localStorage 캐시를 병행**(즉시 표시 + 오프라인 폴백)하되, **정본은 서버**로 한다.

#### 📊 데일리 평가/리포트 · 프로젝트 리포트 — 서버 보관 (자동 기록)

브라우저에만 저장돼 유실되던 데이터를 모두 **서버(DB)에 기록**한다. (localStorage는 캐시로만 병행)

- **AI 데일리 평가**: 업무 종료(🔚) 시 생성되는 한줄평 + AI 평가를 `DailyReport`(사용자·날짜당 1개)에 저장(`POST /ai/daily-review`가 upsert). 지난일 조회 시 `GET /ai/daily-report?userId=&date=`로 로드 → **캐시 삭제·다른 기기에서도 표시**.
- **프로젝트 리포트**(요약/이슈/다음계획): `Project.reportNotes`(JSON)에 저장, **팀 공유**. 리포트 모달에서 입력 시 0.8초 디바운스 자동저장(`PATCH /projects/:id`) + 닫을 때 저장. (사용자가 실제 편집했을 때만 저장해 빈 값 덮어쓰기 방지)

#### 🆕 최상단: ✅ 오늘 할일 체크리스트

오늘 수행할 태스크가 체크리스트 형태로 가장 먼저 표시된다. **체크리스트가 곧 업무의 시작/종료 트리거**가 되어 대시보드 및 근무 로그와 실시간 연동된다.

```
오늘 할일
┌────────────────────────────────────────────────────┐
│ ☐ 메인 히어로 개발        📊🎥   [시작]            │
│ ▶ 프론트 리팩토링         —      ⏱ 10:00~ [종료]   │  ← 진행 중
│ ☑ 디자인 토큰 정리        📊     09:00~09:40 ✓     │  ← 완료
└────────────────────────────────────────────────────┘
```

**동작 흐름:**

| 액션 | 결과 |
|---|---|
| 체크박스 **체크 (시작)** | • 해당 업무가 **대시보드에서 즉시 "진행 중"으로 시작**<br>• **시작 시각 자동 기록** (`Task.startedAt`)<br>• 내 상태가 🟢 업무중 + 현재 태스크로 갱신<br>• 스케줄 타임라인에 진행 블록 생성 |
| **종료 버튼** 클릭 | • 업무 **종료 처리**, **종료 시각 자동 기록** (`Task.endedAt`)<br>• 시작~종료 구간이 타임라인/근무 로그에 정리됨<br>• 업무 종류에 따라 **산출물 입력 폼 노출** (아래 참조) |

**🔁 '진행중'은 하루 단위 — 다음날 자동 리셋:**

- "현재 업무중(진행중)"은 **오늘 시작/재개한 업무**만 표시한다. 즉 진행중 상태(`doing`)는 하루 단위로 관리된다.
- 퇴근(🔚 업무 종료) 또는 개별 종료를 누르지 않고 **탭만 닫고 다음날 다시 접속**하면, 전날부터 열려 있던 진행중 업무는 **자동으로 '중단(paused)'으로 내려가** "현재 업무중"에서 빠진다. (앱 진입 시 `POST /tasks/day-reset` 실행 — 기준: 한국시간(KST) 자정. 열린 근무로그는 시작일 근무종료시각으로 닫아 밤샘 과다집계 방지)
- 이렇게 내려간 업무는 "오늘의 업무"에 **「진행중이던 업무」 배지**로 표시되며, **다시 체크(재개)하면** 오늘의 "현재 업무중"으로 다시 올라온다.
- 참고: 퇴근(🔚 업무 종료) 시에도 진행중 업무는 모두 '중단' 처리된다(기존 동작).

**🆕 종료 시 산출물 입력 (업무 종류 연동):**

종료를 누르면, 부여 시 지정된 [업무 종류]에 맞춰 입력 폼이 뜬다.

| 업무 종류 | 종료 시 노출되는 입력 |
|---|---|
| 📊 보고링크 필요 | 보고 링크 입력란 |
| 🎥 설명영상 필요 | 설명영상 링크/파일 업로드 |
| 📊🎥 둘 다 필요 | 보고 링크 + 설명영상 둘 다 |
| — 둘 다 불필요 | 입력 없이 바로 종료 |

- 업무 설명 작성이 필요한 경우, 종료 시점에 **간단한 업무 설명을 직접 작성**하거나 **링크로 대체** 입력할 수 있다.
- 산출물 미입력 시(둘 다 불필요 제외) "완료 대기" 상태로 남고, 입력 완료 시 ✅ 완료로 전환된다.
- 종료 후에는 시작 시각, 종료 시각, 소요 시간, 산출물 링크가 한 카드로 **자동 정리**되어 기록된다.

## 5. AI 시스템

### 5-1. AI 소통 요약 (프로젝트)

#### 동작 흐름

```
프로젝트 대화 누적
        │
        ▼
┌─── 트리거 조건 ────────────┐
│ • 메시지 20건 이상 누적     │
│ • 매일 오전 9시 정기 실행   │
│ • 수동 "지금 요약" 버튼     │
└────────────┬───────────────┘
             ▼
   BullMQ 작업 큐에 등록
             │
             ▼
   Worker가 해당 프로젝트의 최근 대화 컨텍스트 수집
             │
             ▼
   LLM API 호출
             │
             ▼
   요약 결과를 Project.aiSummary에 저장
   → 프로젝트 페이지 상단에 실시간 반영
```

#### AI 요약 프롬프트 구조

```
시스템: 당신은 프로젝트 관리 어시스턴트입니다.
        다음 프로젝트 대화 내용을 분석하여 요약하세요.

요약 포맷:
  1) 핵심 결정사항 — 팀이 합의한 내용
  2) 진행 상황    — 현재까지 완료된 작업
  3) 미결 이슈    — 아직 결정되지 않은 사항, 필요한 액션아이템

입력: [프로젝트 대화 내용]
```

#### AI 요약 표시 위치

- 프로젝트 상세 페이지 **커뮤니케이션 스레드 상단**에 접이식 카드로 표시
- 마지막 갱신 시간 표시
- "지금 요약" 버튼으로 수동 갱신 가능

### 🆕 5-2. AI 업무설명 doc 생성 (업무 부여)

부여자의 간략한 메모를 정돈된 업무설명 doc으로 변환한다.

#### 동작 흐름

```
부여자 간략 메모 입력
        │
        ▼
정리 프롬프트 적용 (기본 제공, 부여자 수정 가능)
        │
        ▼
BullMQ 큐 등록 → Worker → LLM API 호출
        │
        ▼
업무설명 doc 생성 (배경/목표/범위/산출물/체크포인트)
        │
        ▼
부여자에게 미리보기 전달 (검토 단계)
   ├─ 업무 설명 직접 수정
   └─ 프롬프트 수정 후 재생성
        │
        ▼
확정 → Task.aiDescriptionDoc 저장 → 담당자에게 첨부 부여
```

#### 기본 정리 프롬프트 구조 (수정 가능)

```
시스템: 당신은 업무 정의 어시스턴트입니다.
        아래 간략 메모를 바탕으로 담당자가 바로 이해하고
        착수할 수 있는 업무설명 문서를 작성하세요.

출력 포맷:
  1) 배경 / 목적
  2) 목표 (완료 기준)
  3) 작업 범위
  4) 요구 산출물 (업무 종류에 맞춰: 보고링크 / 설명영상)
  5) 체크포인트 / 마감

입력: [부여자 간략 메모], [업무 종류], [마감일]
```

> 부여자는 위 프롬프트와 생성된 본문을 모두 수정할 수 있으며, 수정 후 재생성하면 새 doc으로 갱신된다.

### v1.2 (2026-06-26) — 업무 대분류 & 프로젝트 인원 구성

**1. 업무 대분류 체계 신설 (§2 · 2-1 B)**
- 모든 업무를 3가지 대분류로 구분: 🎬 롱 / ⚡ 쇼츠 / 📁 프로젝트
- 롱·쇼츠: 대시보드 [업무 부여]에서 담당자 지정해 직접 부여
- 프로젝트: 대시보드에서 직접 부여 불가 → **프로젝트 탭에서 생성**으로 안내
- 대분류와 산출물 요구(보고링크/설명영상)는 별개 축으로 조합 가능

**2. 프로젝트 생성·설정 플로우 (2-3)**
- `+ 새 프로젝트` → 3단계 설정 마법사(기본 설정 → 인원 구성 → 관련 데이터)
- **담당자(Owner, 역할별 책임자)** 와 **참여자(Participant, 협업 멤버)** 구분 및 권한 정의
- 생성 후 상세 페이지에서 인원 추가/제거, 초대 알림

**데이터 모델**
- Task: `category (long / shorts / project)` 추가, project 연결 조건 명시
- Project: `members[]` → **`owners[]`(담당자) + `participants[]`(참여자)** 로 분리

### v1.1 (2026-06-26) — 신규 기능 3종 반영
